# Home SOC Lab — Wazuh SIEM & Attack Simulation

A home Security Operations Center lab built from scratch using open-source tools. This project covers SIEM deployment, endpoint monitoring, attack simulation, and incident response documentation.

## Phases

### Phase 1 — Infrastructure
- Deployed Wazuh 4.14.5 full stack on Ubuntu Server using the official install assistant
- Configured SSL certificates for all inter-component communication
- Fixed post-reboot issues: `vm.max_map_count`, missing `/var/log/wazuh-indexer/` directory
- Enrolled Windows 10 agent via auto-enrollment on port 1515
- Configured static IPs on the isolated internal network (no DHCP server)

### Phase 2 — Endpoint Visibility
- Installed Sysmon on Windows 10 with SwiftOnSecurity config
- Added Sysmon event channel to Wazuh agent `ossec.conf`:
```xml
<localfile>
    <location>Microsoft-Windows-Sysmon/Operational</location>
    <log_format>eventchannel</log_format>
</localfile>
```
- Verified Sysmon telemetry flowing into Wazuh dashboard

### Phase 3 — Attack Simulation
Attacks executed from Kali Linux against the Windows 10 endpoint:

| Attack | Tool | MITRE Technique |
|---|---|---|
| Port scanning | Nmap `-sV -O` | T1046 — Network Service Scanning |
| Service enumeration | Nmap `-sC -A` | T1046 — Network Service Scanning |
| SMB enumeration | enum4linux | T1087 — Account Discovery |
| Brute force simulation | PowerShell script | T1110 — Brute Force |

### Phase 4 — Detection & Documentation
- Wazuh detected all attack phases in real time
- 386 total security events captured
- 73 high-severity alerts (Level 12+)
- Full MITRE ATT&CK mapping auto-generated by Wazuh
- Incident response report written (see `/docs/`)

## Lessons Learned

- OpenSearch requires `vm.max_map_count=262144` — must be set permanently in `/etc/sysctl.conf`
- VirtualBox internal networks have no DHCP server — static IP assignment required on all VMs
- Wazuh Indexer needs `/var/log/wazuh-indexer/` to exist on boot — fixed via systemd tmpfiles
- Sysmon dramatically increases detection visibility beyond default Windows Event Logs
- The Wazuh agent enrollment port (1515) must be reachable from the endpoint — network connectivity is the first thing to check when agents fail to register
